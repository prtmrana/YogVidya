Before OOPS (procedural programming):

-  Code duplication ❌ 
-  Hard to maintain ❌ 
-  No structure ❌ 
OOPS provides:

-  Reusability ✅ 
-  Maintainability ✅ 
-  Scalability ✅ 
-  Security (data hiding) ✅


OOPS Principle:

1. Encapsulation : Wrapping **data (variables) + behavior (methods)** together in a class and restricting direct access.
 How to achieve it :

- [ ]  Use `private` variables 
- [ ]  Provide `public`  getters/setters
Example : String /Wrapper classes



2 . **Abstraction** → Hiding implementation and exposing only functionality

How to achieve it :

- [ ]  Use Interface 
- [ ]  User Abstract class
Example : 1. set/list/map

                  2. abstractList / abstractSet﻿  / HttpSession
